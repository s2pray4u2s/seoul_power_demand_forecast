### 📌 변수 설명

#### 🌡️ CDD (Cooling Degree Day, 냉방도일)
> "오늘 얼마나 더웠는가"를 숫자로 표현한 값

기준 온도 18°C보다 기온이 높을수록 에어컨을 더 많이 켬<br>
예를 들어 오늘 기온이 28°C라면 CDD = 28 - 18 = **10**<br>
CDD가 높을수록 냉방 전력 수요가 올라간다고 보면 됨.

```
CDD = max(기온 - 18, 0)
더울수록 CDD 높음 → 전력 수요 높음
```

---

#### 🌨️ HDD (Heating Degree Day, 난방도일)
> "오늘 얼마나 추웠는가"를 숫자로 표현한 값

기준 온도 18°C보다 기온이 낮을수록 난방을 더 많이 켬<br>
예를 들어 오늘 기온이 3°C라면 HDD = 18 - 3 = **15**<br>
HDD가 높을수록 난방 전력 수요가 올라감

```
HDD = max(18 - 기온, 0)
추울수록 HDD 높음 → 전력 수요 높음
```

---

#### ⏰ lag_24 (24시간 전 수요)
> "어제 이 시간에 전기를 얼마나 썼는가"

전력 수요는 관성이 있음<br>
어제 오후 2시에 전기를 많이 썼다면, 오늘 오후 2시에도 비슷하게 많이 쓸 가능성이 높다고 확인<br>
이 패턴을 모델에 학습시키기 위해 사용

```
lag_24 = 24시간 전 전력사용량
→ 어제와 오늘의 패턴이 비슷하다는 가정
```

---

#### 📅 lag_168 (168시간 = 1주일 전 수요)
> "지난주 같은 요일 이 시간에 전기를 얼마나 썼는가"

사람들의 생활 패턴은 매주 비슷<br>
지난주 월요일 오전 9시와 이번주 월요일 오전 9시의 전력 수요는 유사한 경향이 있음<br>

```
lag_168 = 168시간(7일) 전 전력사용량
→ 지난주 같은 요일·시간 패턴 반영
```

---

#### 📊 rolling_24h (직전 24시간 이동평균)
> "최근 하루 동안 평균적으로 전기를 얼마나 썼는가"

최근 24시간 평균이 높으면 지금도 높은 수요 구간일 가능성이 크다는 의미

```
rolling_24h = 직전 24시간 전력사용량 평균
→ 최근 추세 반영
```

---

#### 🌬️ 체감온도
> "사람이 실제로 느끼는 온도"

같은 기온이라도 바람이 강하면 더 춥게 느껴지기 마련<br>
겨울철에는 기온뿐 아니라 풍속까지 고려한 체감온도가 난방 수요를 더 잘 설명이 가능<br>
기상청 공식 기준(기온 10°C 이하 & 풍속 1.3m/s 이상)일 때만 적용

```
겨울 체감온도 = 13.12 + 0.6215T - 11.37V^0.16 + 0.3965TV^0.16
(T: 기온, V: 풍속 km/h)
조건 미충족 시 → 기온 그대로 사용
```

---

#### 😓 불쾌지수 (DI, Discomfort Index)
> "여름에 얼마나 불쾌하게 더운가"

기온이 같아도 습도가 높으면 더 덥게 느껴지기 마련<br>
불쾌지수는 기온 + 습도를 합쳐서 체감 더위를 수치화<br>

```
DI = 0.81 × 기온 + 0.01 × 습도 × (0.99 × 기온 - 14.3) + 46.3

DI 75 이상 → 냉방 필요
DI 80 이상 → 열 스트레스 심화 (냉방 피크 위험)
```

---

#### ☀️ 태양광추정
> "태양광 발전소가 이 시간에 얼마나 전기를 만들었을까"

실제 태양광 발전량 데이터가 없어서 일사량 데이터로 추정한 값

```
태양광추정 = 일사량(MJ/m²) × 0.2778 × 0.18
(0.2778: MJ→MWh 단위변환, 0.18: 효율 계수)
※ 실제 발전량이 아닌 추정값
```

### 3. ML 모델링 — 4개 모델 비교 (XGBoost 기준)

> **모델 구성 설명**
> - **Baseline**: 시간/요일만으로 예측 → "아무것도 모르는 상태"
> - **LAG Model**: 과거 전력량 패턴 추가 → "어제·지난주 수요 참고"
> - **ENV Model**: 환경 변수만으로 예측 → "날씨 정보만으로 예측"
> - **Full Model**: 모든 변수 사용 → "날씨 + 과거 패턴 모두 활용"

| 모델 | RMSE | R² | MAPE |
|------|------|-----|------|
| Baseline | 651.8 | 0.8126 | 8.75% |
| LAG Model | 254.2 | 0.9715 | 2.89% |
| ENV Model | 477.7 | 0.8993 | 7.59% |
| Full Model | 256.8 | 0.9709 | 2.86% |

> **지표 설명**
> - **RMSE**: 예측 오차 MWh (낮을수록 좋음)
> - **R²**: 설명력 (1.0에 가까울수록 좋음)
> - **MAPE**: 평균 오차율 % (낮을수록 좋음)

**핵심 발견**
- ENV Model: 날씨만으로 **R² 0.90 달성** → 환경 변수 독립적 예측력 보유
- 고온 극단 구간에서 Full Model이 LAG 대비 **RMSE 12.9% 개선**
- 평소엔 lag(관성)가 지배적이지만 **폭염·한파 극단 구간에서 환경 변수가 결정적**

### 4. 탄소 배출량 연계 (인사이트 활용 예정)

> 전력을 생산하려면 화력발전소를 돌려야 하고, 그 과정에서 탄소가 배출됨
> 전력 수요를 정확히 예측하면 불필요한 발전을 줄여 탄소도 감축 가능

```
전력사용량(MWh) × 0.4541 = 탄소배출량(tCO2eq)
(2024년 승인 국가 온실가스 배출계수 — 소비단 기준)
```

| 구분 | 평균 수요 | 시간당 탄소 배출 |
|------|----------|----------------|
| 일반일 | 4,896 MWh | 약 2,224 tCO2eq |
| 폭염일 | 7,676 MWh | 약 3,487 tCO2eq |
| **차이** | **+2,780 MWh** | **+1,263 tCO2eq** |

> 💡 폭염 1시간 추가 탄소 배출 1,263톤 = 나무 약 19만 그루가 1년간 흡수하는 양

### 🔎 해석: 환경 변수의 역할

모델 비교 결과, 과거 수요 패턴(lag)은 평시 전력 수요 예측에 매우 강력한 설명력을 보였으나,
폭염 및 한파와 같은 급격한 환경 변화 상황에서는 한계를 보임

특히, 고온 극단 구간에서 Full Model이 LAG 대비 RMSE를 개선한 점은
환경 변수가 단순 설명 변수가 아닌 **이상기온 대응 변수**로 작동함을 시사

즉,

- lag 변수 → 평소 패턴 설명
- 환경 변수 → 극단 상황 보정

이라는 역할 구분이 확인되었다.

### 🌦️ 계절별 환경 변수 적용 전략

환경 변수는 계절에 따라 영향력이 다르게 작용하므로
단일 변수 접근보다 계절 맞춤형 접근이 필요

| 계절 | 주요 설명 변수 | 적용 목적 |
|------|---------------|-----------|
| 여름 | 불쾌지수(DI) | 냉방 수요 및 폭염 대응 |
| 겨울 | 체감온도 | 난방 수요 설명 |
| 중간계절 | CDD / HDD | 완만한 냉난방 수요 반영 |

이는 단순 기온 기반 모델보다
"인간 체감 기반 전력 수요 반응"을 반영하기 위한 접근

---

### ⚡ Lag vs 환경 변수 관계 정리

Lag 변수는 수요의 관성을 설명
그러나 관성은 환경 변화의 속도를 반영하지 못함

예:

- 갑작스러운 폭염 시작
- 장마 후 급격한 습도 상승
- 강풍 동반 한파

이러한 상황에서는

과거 수요 ≠ 현재 수요

가 발생하며,
이때 환경 변수의 설명력이 증가

---

### 🎯 모델링 방향성

현재 결과를 바탕으로 모델의 해석은 다음과 같다.

- Lag Model → 평시 예측 최적
- ENV Model → 환경 반응 설명 가능
- Full Model → 극단 상황 대응력 확보

따라서 본 프로젝트는

"패턴 기반 예측 + 환경 기반 보정"

이라는 이중 구조 접근을 지향

---

### 🌱 확장 가능성 (탄소 관점)

환경 기반 수요 변화는 단순 전력 소비 문제가 아니라
탄소 배출 변동성과 직접 연결

특히 폭염 상황에서

수요 증가 → 발전 가동 증가 → 탄소 배출 증가

로 이어질 수 있으므로,
환경 변수 반영 모델은
향후 탄소 배출 예측 및 운영 전략 수립에도 활용 가능

---

### 🧭 현재 팀의 분석 포지션

팀의 접근은 단순한 수요 예측이 아닌,

환경 변화에 반응하는 전력 수요 구조를 이해하는 모델을 구축하는 목표

이는 향후

- 피크 대응
- 탄소 운영
- 수요관리 전략

으로 확장 가능한 기반 분석
